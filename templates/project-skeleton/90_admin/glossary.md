# 용어집 (Glossary)

이 문서는 프로젝트 전체에서 사용되는 용어의 **공식 정의**입니다.

## 사용 목적

1. 문서 작성 시 일관된 용어 사용
2. 번역 시 용어 통일 (ko-KR ↔ en-US)
3. 팀 간 커뮤니케이션 명확화

---

## 시스템 용어

| 한국어 (ko-KR) | 영어 (en-US) | 정의 | 비고 |
|----------------|--------------|------|------|
| 사용자 | User | 시스템을 이용하는 최종 사용자 | |
| 관리자 | Admin | 시스템 관리 권한을 가진 사용자 | Administrator 아님 |
| 인증 | Authentication | 사용자 신원 확인 과정 | Auth로 축약 가능 |
| 인가 | Authorization | 권한 확인 및 부여 과정 | |
| 세션 | Session | 인증된 사용자의 연결 상태 | |
| 토큰 | Token | 인증 정보를 담은 문자열 | |

## 도메인 용어

| 한국어 (ko-KR) | 영어 (en-US) | 정의 | 비고 |
|----------------|--------------|------|------|
| (프로젝트별 정의) | | | |

## 기술 용어

| 한국어 (ko-KR) | 영어 (en-US) | 정의 | 비고 |
|----------------|--------------|------|------|
| 엔드포인트 | Endpoint | API 접점 URL | |
| 스키마 | Schema | 데이터 구조 정의 | |
| 페이로드 | Payload | 요청/응답 본문 데이터 | |
| 미들웨어 | Middleware | 요청 처리 중간 계층 | |

## Byeori 용어

| 한국어 (ko-KR) | 영어 (en-US) | 정의 |
|----------------|--------------|------|
| 초안 | Draft | 작업 중인 문서 상태 |
| 리뷰 | Review | AI Agent의 검토 결과 |
| 승인 | Approval | Human의 최종 확인 |
| 버전 | Version | 승인된 문서의 스냅샷 |
| 릴리스 | Release | 개발 착수 가능한 최종 번들 |
| 수용 기준 | Acceptance Criteria | Task 완료 판단 기준 |

---

## 용어 추가 규칙

1. 새 용어 추가 시 한국어/영어 모두 정의
2. 약어 사용 시 전체 명칭 함께 기록
3. 도메인 특화 용어는 "도메인 용어" 섹션에 추가
4. 번역 시 이 용어집을 **반드시** 참조
