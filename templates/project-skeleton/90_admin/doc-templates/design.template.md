## 6. 에러 처리

### 6.1 에러 분류

| 카테고리 | 코드 범위 | 설명 | HTTP 상태 |
|----------|-----------|------|-----------|
| Validation | 1000-1999 | 입력 검증 실패 | 400 |
| Authentication | 2000-2999 | 인증 실패 | 401 |
| Authorization | 3000-3999 | 권한 없음 | 403 |
| NotFound | 4000-4999 | 리소스 없음 | 404 |
| Business | 5000-5999 | 비즈니스 규칙 위반 | 422 |
| System | 9000-9999 | 시스템 오류 | 500 |

### 6.2 에러 응답 형식

```json
{
  "error": {
    "code": "ERR-1001",
    "message": "사용자에게 표시할 메시지",
    "details": {
      "field": "email",
      "reason": "형식이 올바르지 않습니다"
    },
    "timestamp": "2026-03-07T10:00:00Z",
    "traceId": "abc-123-def"
  }
}
```

### 6.3 에러 처리 전략

| 에러 유형 | 처리 방식 | 재시도 | 알림 |
|-----------|-----------|--------|------|
| 일시적 오류 | 재시도 | 3회 | ❌ |
| 영구적 오류 | 즉시 반환 | ❌ | ❌ |
| 시스템 오류 | 로깅 + 알림 | 상황별 | ✅ |
