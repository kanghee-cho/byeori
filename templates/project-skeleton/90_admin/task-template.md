## Acceptance Criteria (AC)

> **Principle**: A task is valid only when AC are complete and testable

### AC-1: (Happy Path)
**Given** (Precondition / Initial state)  
**When** (Action / Event / Request)  
**Then** (Observable result)

### AC-2: (Validation Failure)
**Given** (Invalid input or prohibited state)  
**When** (Action / Event / Request)  
**Then** (Error result + Error type / Message / Status)

### AC-3: (Edge Case / Error)
**Given** (Boundary condition)  
**When** (Action)  
**Then** (Expected result)
